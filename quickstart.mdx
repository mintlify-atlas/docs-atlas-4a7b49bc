<Steps>

<Step title="Install pygame-ce and clone PycePlus">

PycePlus requires `pygame-ce`. Do **not** install the original `pygame` — the two packages conflict.

```bash
pip install pygame-ce
git clone https://github.com/GrandKenzy/PycePlus.git
```

If PycePlus ships a `requirements.txt`, install it too:

```bash
cd PycePlus && pip install -r requirements.txt
```

<Tip>
  Add the cloned `PycePlus` directory to your `PYTHONPATH`, or place your app file at the same level as the `PycePlus/` folder so the import resolves without extra configuration.
</Tip>

</Step>

<Step title="Create a window">

`window.new()` initializes the Pygame display and stores it internally. You call it once before the loop starts.

```python app.py
import PycePlus as pp

win = pp.window.new(
    "My App",
    width=800,
    height=600,
    background_color=(20, 20, 30),
)
```

`window.new()` signature (from `UI/window.py`):

| Parameter          | Type                      | Default              | Description                              |
| ------------------ | ------------------------- | -------------------- | ---------------------------------------- |
| `title`            | `str`                     | required             | Window title bar text                    |
| `width`            | `int`                     | screen width − 200   | Window width in pixels                   |
| `height`           | `int`                     | screen height − 200  | Window height in pixels                  |
| `x`                | `int`                     | `CENTER` (−1)        | Horizontal position; −1 centres the window |
| `y`                | `int`                     | `CENTER` (−1)        | Vertical position; −1 centres the window |
| `background_color` | `tuple[int, int, int]`    | `(0, 0, 0)`          | RGB fill colour applied each frame       |
| `resizable`        | `bool`                    | `False`              | Allow the user to resize the window      |
| `fulls_screen`     | `bool`                    | `False`              | Start in full-screen mode                |
| `vsync`            | `bool`                    | `False`              | Enable vertical sync                     |

</Step>

<Step title="Add a Frame widget">

`Frame` is a rectangular container that renders its children and supports mouse events, borders, padding, and optional scrolling.

```python app.py
import PycePlus as pp

win = pp.window.new("My App", width=800, height=600, background_color=(20, 20, 30))

panel = pp.Frame(
    width=300,
    height=200,
    background_color=(80, 40, 200),
    x=100,
    y=100,
)
```

Key `Frame` parameters (from `GUI/frame.py`):

| Parameter          | Type                   | Default     | Description                         |
| ------------------ | ---------------------- | ----------- | ----------------------------------- |
| `width`            | `int`                  | `200`       | Frame width in pixels               |
| `height`           | `int`                  | `200`       | Frame height in pixels              |
| `background_color` | `colorType`            | `(0, 0, 0)` | Fill colour                         |
| `x`                | `int`                  | `0`         | Horizontal position inside parent   |
| `y`                | `int`                  | `0`         | Vertical position inside parent     |
| `border_width`     | `int`                  | `0`         | Outline stroke width                |
| `border_color`     | `colorType`            | `(0, 0, 0)` | Outline colour                      |
| `border_radius`    | `int`                  | `0`         | Corner radius                       |
| `scrollable`       | `bool`                 | `False`     | Enable scroll behaviour             |
| `page`             | `str \| None`          | `None`      | Assign to a named page              |
| `name`             | `str \| None`          | `None`      | Widget identifier for `Tree.nav()`  |

<Note>
  Every `Frame` created without a `father` argument is added to the global widget tree automatically. You do not need to register it manually.
</Note>

</Step>

<Step title="Bind a click event">

Call `bind()` on any widget to attach a callback. Event names follow a dot-separated convention: `mouse.button.left.click`, `mouse.hover`, `mouse.leave`, etc.

```python app.py
def on_click():
    print("panel clicked")

panel.bind("mouse.button.left.click", on_click)
```

All widget events available (from `GUI/base.py`):

| Event string                      | Fires when                          |
| --------------------------------- | ----------------------------------- |
| `mouse.button.left.click`         | Left mouse button pressed down      |
| `mouse.button.left.release`       | Left mouse button released          |
| `mouse.button.left.repeat`        | Left button held (fires every frame)|
| `mouse.button.middle.click`       | Middle button pressed down          |
| `mouse.button.middle.release`     | Middle button released              |
| `mouse.button.right.click`        | Right button pressed down           |
| `mouse.button.right.release`      | Right button released               |
| `mouse.hover`                     | Cursor enters the widget area       |
| `mouse.leave`                     | Cursor leaves the widget area       |
| `mouse.hovering`                  | Cursor is over the widget (per frame)|

`bind()` returns `self`, so you can chain it:

```python app.py
panel.bind("mouse.hover", lambda w, pos: print("hovered")).bind("mouse.leave", lambda w, pos: print("left"))
```

</Step>

<Step title="Start the loop">

`pp.mainloop()` enters the `while True` render loop. Call it after all setup is complete — it does not return until the window is closed.

```python app.py
pp.mainloop(FPS=60)
```

`mainloop()` parameters (from `CORE/main.py`):

| Parameter          | Type                        | Default | Description                                               |
| ------------------ | --------------------------- | ------- | --------------------------------------------------------- |
| `FPS`              | `int`                       | `60`    | Target frames per second                                  |
| `CALL_PER_SECOND`  | `Callable \| None`          | `None`  | Function called once per second (network/resource checks) |
| `REVIEW_PATHS`     | `list[str]`                 | `[]`    | Extra directories to watch for file changes               |
| `REVIEW_MAIN_DIR`  | `bool`                      | `True`  | Watch the app's own directory for file changes            |

</Step>

</Steps>

## Full working example

```python app.py
import PycePlus as pp

# 1. Create window
win = pp.window.new(
    "My First PycePlus App",
    width=800,
    height=600,
    background_color=(20, 20, 30),
)

# 2. Create a Frame widget
panel = pp.Frame(
    width=300,
    height=200,
    background_color=(80, 40, 200),
    x=100,
    y=100,
)

# 3. Bind mouse events
panel.bind("mouse.button.left.click", lambda w, pos: print("clicked!", pos))
panel.bind("mouse.hover", lambda w, pos: print("hovering"))

# 4. (Optional) React to system events
pp.events.register_event("system.idle", lambda t: print(f"idle at {t:.2f}s"))

# 5. Start the loop
pp.mainloop(FPS=60)
```

Run it with:

```bash
python app.py
```

You should see an 800×600 dark window with a purple rectangle. Clicking the rectangle prints `clicked!` to the terminal.

<Tip>
  Pass `CALL_PER_SECOND=my_function` to `mainloop()` to run logic exactly once per second — useful for polling a server or updating a timer display.
</Tip>
